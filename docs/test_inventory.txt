Agri Platform Test Inventory
Last updated: 25 March 2026

Purpose
This file records every automated test in the project and explains, in plain language, what each one proves. It is intended as a maintenance reference rather than an executable artifact.

Test Structure
- tests/pages: route and HTML-response behavior
- tests/logic: pure logic, logging, and PDF generation
- tests/database: schema, generated data, helper behavior, and persistence

tests/pages/test_settings.py

- test_settings_page_returns_success
  Confirms the settings page loads successfully and renders the input catalog management interface.
- test_settings_page_can_add_catalog_entry
  Confirms a new standardized input can be created through the settings form and persisted to the database.
- test_settings_page_deactivates_used_catalog_entry
  Confirms a catalog item already referenced by farm input logs is marked inactive rather than deleted.
- test_settings_page_can_reset_database
  Confirms the reset action rebuilds the demo database and restores the seeded buyer-account count.

- test_buyer_pledges_page_renders_progress_for_in_progress_pledges

- test_buyer_pledges_page_renders_empty_state_when_filters_remove_all_rows
